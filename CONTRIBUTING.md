# 贡献指南

本仓库定位为公开营销官网，请尽量保持内容表达、SEO 结构和联系表单链路清晰。

## 1. 基本原则

- 官网负责公开内容，不承担后台业务逻辑
- 所有对外接口调用优先放入 `app/composables`
- 页面文案可替换，但结构应保持可复用
- 涉及 SEO 的改动尽量同步更新 meta 描述

## 2. 推荐开发流程

1. 拉取最新代码
2. 创建分支
3. 本地预览页面效果
4. 运行类型检查与构建
5. 提交并发起 PR

## 3. 提交前检查

```powershell
npm run type-check
npm run build
```

## 4. 页面设计建议

- 优先使用可复用组件而不是重复布局
- 页面视觉需兼顾桌面端与移动端
- 联系表单字段如果变化，要同步后端契约

## 5. 提交规范

请参考 [`COMMIT_CONVENTION.md`](./COMMIT_CONVENTION.md)
