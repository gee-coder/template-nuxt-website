# 发布流程

## 1. 版本建议

建议使用语义化版本：

- `MAJOR`：破坏性结构或运行时契约变更
- `MINOR`：新增向后兼容的页面或模板能力
- `PATCH`：缺陷修复、文档更新、构建与样式微调

## 2. 发布前检查清单

### 必做检查

- 更新 [CHANGELOG.md](./CHANGELOG.md)
- 核对 README、环境变量说明与部署步骤
- 确认联系表单接口地址与运行时配置一致

### 本地校验命令

```powershell
npm ci
npm run type-check
npm run build
```

## 3. 建议的发布步骤

1. 合并改动到 `master`
2. 补齐 `CHANGELOG.md`
3. 执行类型检查与生产构建
4. 创建标签，例如 `v0.1.0`
5. 在 GitHub Release 中记录升级说明、配置变更与已知限制

## 4. 发布说明建议包含

- 页面与内容模块的新增或调整
- SEO、运行时配置或接口约定变化
- 部署方式是否有变化
- 升级注意事项与已知限制
